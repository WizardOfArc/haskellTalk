Outline

  • What is Haskell?

    Hakell is a pure - functional programming language named after Haskell Curry (one of the developers of lambda calculus)

  • Why should we care?

    • It was one of the main influences on the design of Scala
    • Haskell's strong typing and functionally purity forces one to think in terms of composing deterministic functions.
      This makes one approach Scala in a more functional way—chaining transformations.

  • Why Azí?

    • Not a Haskell expert
    • Software Engineer with a Degree in Math and a love for languages

  • Why Azí loves Haskell?

    • Syntax and approach is inspired by Mathematics

    • Functions are pure and deterministic:

      • Functions are statically and strongly typed
      • Same input will have the same output - always
    • Pattern matching and guards
    • Internal definitions with either "let" or "where"
    • Currying / partial applictaion built in (good for reuse and modularity)
    • No for loops, everything is a function call. List processing functions are abstractions of some recursive calls.
    • Declarative... list of definitions as opposed to a list of instructional steps
    • Type aliasing
    • Non deterministic code (like IO) is quarantined off into monads (a deterministic workaround to handle the non-deterministic world of states)

  • Now what?

    1. Think functionally first - sketch things in Haskell
    2. Porting Haskell patterns to Scala
    3. Separate your IO into isolated, small parts
